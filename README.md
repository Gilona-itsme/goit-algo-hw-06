Завдання 1. Створення та аналіз графа реальної мережі

Було створено граф, який моделює схему Київського метрополітену з трьома лініями:
	•	М1 (червона)
	•	М2 (синя)
	•	М3 (зелена)

Для графа:
	•	Створено вершини та ребра для кожної лінії.
	•	Додано пересадки між станціями.
	•	Побудовано автоматичну візуалізацію з позиціонуванням станцій та кольоровим позначенням ліній.


Визначено вершини з найбільшою центральністю за ступенем — це точки пересадок:
Проведено пошук найкоротшого шляху між двома станціями за кількістю перегонів.

Завдання 2. DFS та BFS у графі метро

Було реалізовано власні алгоритми DFS та BFS, що повертають конкретний шлях між двома станціями.

1.DFS 

Повертає перший знайдений шлях, рухаючись якомога глибше вздовж гілки.
Через це шлях зазвичай довший та менш оптимальний.

2.BFS

Шукає рівнями і повертає найкоротший шлях за кількістю ребер, тобто мінімальна кількість пересадок/перегонів.

DFS не оптимізує шлях — досліджує гілку до кінця, тому результат залежить від порядку сусідів.
BFS гарантує найкоротший шлях, бо проходить граф рівнями від старту.
